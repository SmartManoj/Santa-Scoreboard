## üìä Competition Details

- **200 tasks** across the competition (001-200)
- Each task requires optimal placement and rotation of Christmas trees
- Submission file contains 20,100 entries with coordinates (x, y) and rotation degrees
- **Total Score**: 83.031685401301 (calculated using official metric)

## üéØ Metric Details

The competition metric evaluates:
- **Normalized bounding box area** for each task
- Lower scores indicate more compact packing
- Each tree is a polygon shape that can be rotated
- Bounding box must contain the entire rotated tree polygon

## üßÆ Score Calculation

To calculate individual task scores and total score:

```bash
python calculate_scores.py
```

This will:
- Load the submission.csv file
- Group entries by task ID (001-200)
- Calculate scores for each task using the official competition metric
- Account for tree polygon geometry and rotations
- Generate `task_scores.csv` with results
- Display summary statistics with 12 decimal place precision

### Metric Implementation

The script implements the official competition metric:
- **Formula**: score = Œ£(s¬≤/n) for all tasks
- **s** = side length of square bounding box containing all trees
- **n** = number of trees in the configuration
- Uses actual tree polygon geometry (not simple circles)
- Accounts for tree rotations

The metric uses the tree polygon shape with:
- Trunk: 0.15 width √ó 0.2 height
- Base tier: 0.7 width
- Middle tier: 0.4 width
- Top tier: 0.25 width
- Tip at y = 0.8

The website automatically loads and displays scores from `task_scores.csv` when available.
